Abstract
Determining the optimal sequence of compiler optimization
passes is challenging due to the extensive and intricate search
space. Traditional auto-tuning techniques, such as iterative
compilation and machine learning methods, are often limited
by high computational costs and difficulties in generalizing to
new programs. These approaches can be inefficient and may
not fully address the varying optimization needs across dif-
ferent programs. This paper introduces a novel approach that
leverages the synergistic relationships between optimization
passes to effectively reduce the search space. By focusing
on chained synergy pass pairs that jointly optimize a spe-
cific target, our method uses K-means clustering to capture
common optimization patterns across programs and forms
these pairs into coresets. Leveraging a supervised learning
model trained on these coresets, we effectively predict the
most beneficial coreset for new programs, streamlining the

1 Introduction
Compilers offer a range of optimization passes to enhance
program performance by reducing the consumption of re-
sources such as memory, CPU, and storage. Determining the
optimal sequence of passes manually is challenging, espe-
cially as the number of passes increases. For instance, with
124 passes, the number of possible sequences of length 45
exceeds the number of atoms in the universe [ 22]. To address
this challenge, compiler auto-tuning techniques automate
the process of finding the optimal sequence, significantly
reducing the need for manual optimization.
Considerable progress has been made in improving auto-
tuning techniques. Iterative compilation methods were ef-
fective in the earlier stages of compiler optimization, while
recent machine learning techniques have been leveraged
to guide the search process [ 7]. These include performance
modeling to accelerate iterations, direct machine learning ap-
proaches to predict pass sequences, and reinforcement learn-
ing techniques to find optimal pass sequences [ 4,7,21,27].
Despite the advancements, existing methods have limita-
tions. For example, search-based tools like Opentuner [ 3],
BOCA [ 9], CompTuner [ 28], and TPE [ 6] focus on finding
optimal sequences for individual programs, which can be
time-consuming. On the other hand, Autophase [ 18] uses re-
inforcement learning to generalize across programs by learn-
ing different program representations, allowing it to reason
about promising sequences for unseen programs. However,
this approach relies on well-designed program representa-
tions and model structures, and may not perform as well for
unseen programs compared to direct search methods.
In this paper, we introduce a novel approach for finding
the optimal pass sequence by leveraging the interactions
between optimization passes to optimize a specific target.
Our method focuses on identifying chained synergy pass
pairs that work together to enhance program optimization.
We first define a formula to identify these pairs for each
program in the training set, capturing their cooperative ef-
fects. To generalize these pairs across different programs, we
cluster the chained synergy pass pairs using K-means, form-
ing coresets that represent common optimization strategies.
Building on this foundation, we employ a supervised learning
model to intelligently select the most suitable chained syn-
ergy pass pair set from these coresets for unseen programs in
the test set. By narrowing the search space to these selected
sets, our method effectively guides the search strategy to
derive near-optimal pass sequences. Our approach, which
integrates prediction and search, achieves state-of-the-art
performance across ten datasets with an average reduction
of 7.5% in Intermediate Representation (IR) instruction count
compared to Oz. Furthermore, by constructing a search space
using all chained synergy pass pairs from five training sets,
aimed at reducing codesize, we successfully guided the re-
duction of codesize across the corresponding test sets ofthese datasets. By applying a genetic algorithm that takes
only 6 seconds to this search space, our approach achieves
state-of-the-art codesize reduction results, with an average
reduction of 13.9%, outperforming other search-based meth-
ods applied within the 124-pass search space, such as BOCA,
OpenTuner, and TPE.
In summary, our contributions are as follows:
•We propose a novel method to identify chained syn-
ergy pass pairs that enhance compiler optimization.
The method evaluates the impact of applying combina-
tions of passes on a specific target and identifies pairs
that provide significant additional optimization bene-
fits. This approach facilitates the discovery of potential
optimization opportunities and reduces redundant op-
timization operations.
•We cluster the chained synergy pass pairs identified by
our algorithm for each program. This clustering pro-
cess creates coresets of chained synergy pass pair sets
that capture common optimization strategies, enabling
effective reuse of optimization knowledge across dif-
ferent programs.
•Our approach demonstrates state-of-the-art perfor-
mance in reducing IR instruction count and codesize,
outperforming existing techniques while generalizing
well to unseen programs with minimal time overhead.
•We provide a set of search strategies for the chained
synergy pass pair set, including random walk, greedy
algorithm, genetic algorithm, and reinforcement learn-
ing, with the genetic algorithm (GA) showing the best
performance.
2 Motivation
2.1 Exploring Valuable Search Spaces
When automating the search for an optimal pass sequence
for a program, a significant challenge arises due to the enor-
mous and redundant search space. To address this, if we can
eliminate passes that are not beneficial for the program and
reduce the search space, the complexity of the search can be
significantly reduced. Therefore, we attempt to select passes
that are beneficial (i.e., those that reduce the IR instruction
count) to construct a more focused search space and conduct
a random search experiment for pass sequences. In our ex-
periment, we set the length of the randomly searched pass
sequences to 20. Meanwhile, the complete search space con-
sists of 124 transformation passes from LLVM 10.0.0 𝑜𝑝𝑡.
To minimize data variability, we perform 1,000 searches in
the complete search space, whereas the reduced beneficial
search space only requires 100 searches. The objective of
these experiments is to optimize the percentage reduction
in IR instruction count.

pass. Specifically, at each step within the complete search
space, the program selects the pass that results in the greatest
reduction in IR instruction count under the current state,
continuing until the sequence reaches a length of 20.

3 Related Work
Optimizing compiled programs has been a key area of re-
search, aimed at enhancing performance, reducing codesize,
and improving energy efficiency. In the initial phases of re-
search in this domain, a range of optimization techniques
were explored, such as stochastic search and heuristic algo-
rithms. J. Bergstra et al. explored the effectiveness of stochas-
tic search and introduced a novel model-based sequential
optimization algorithm, TPE [ 6], for tuning deep belief net-
works. Y. Chen et al. proposed RIO [ 10], which involves ran-
domly assigning the enable/disable status of each optimiza-
tion flag in each iteration. Ansel et al. presented OpenTuner
[3], a framework for constructing domain-specific multi-
objective program auto-tuners. This framework introduces
the concept of employing a collection of search techniques
in auto-tuning, which enables the collaborative application
of multiple techniques to identify an optimal solution and
facilitates a more robust search than using a single technique.
U. Garciarena utilized a Genetic Algorithm (GA) [ 16] to opti-
mize compiler flag selection. An initial population of random
flag configurations is iteratively refined through crossover
and mutation operations, with each generation evaluated for
program performance and the fittest configurations selected
for further propagation. J. Chen and colleagues proposed
BOCA [ 9], a new method for identifying the most influential
optimization flags. This method employs a random forest
model to identify influential optimization flags, which are
then used for a comprehensive search, thus improving search
efficiency. M. Zhu et al. proposed CompTuner [ 28], a multiple-
phase learning-based compiler auto-tuning technique that
reduces time costs and enhances performance. Unlike BOCA,
CompTuner builds a prediction model using a diverse dataset
and optimizes the search for effective sequences with an im-
proved particle swarm optimization algorithm.
The studies presented above focused on the search for a
single program, which is a relatively time-consuming process.
On the other hand, to reduce the time spent on searching, re-
searchers have initiated investigations into the direct predic-
tion of near-optimal sequences for a given program through
the utilization of machine learning models. Q. Huang et al.
proposed Autophase [ 18], a framework that utilizes deep

the use of a fixed set of sequences in the coreset imposes
certain limitations on its generalization ability across differ-
ent programs. Another study conducted by S. Park proposed
SRTuner [ 26], a tool that enhances program performance by
analyzing and exploiting synergistic relationships between
passes. It employs multi-stage structures and multi-armed
bandit algorithms to efficiently explore the optimization
space. However, SRTuner requires identifying synergistic
relationships for each individual program, which is a time-
consuming process and lacks generalizability. Additionally,
a study on synergistic relationships between passes, desig-
nated as ICMC [ 23], also proposes the concept of synergism.
If for a given program 𝑃, the pass sequence (𝐴,𝐵)outper-
forms both pass 𝐴and pass𝐵individually, as well as the
sequence(𝐵,𝐴), it indicates that pass 𝐴enhances the effec-
tiveness of pass 𝐵, implying that pass 𝐵depends on pass
𝐴. However, this approach to identifying synergistic pairs
is more restrictive and may result in the exclusion of some
potentially useful pairs. Furthermore, subsequent pruning
operations conducted in this study to identify the optimal
sequence on the constructed synergy graphs may lead to the
final outcome being trapped in a local optimum.
We propose a method for constructing an optimization
search space based on synergy. First, we define a chained
synergy pass pair that is suitable for connecting. Each search
space consists of multiple chained synergy pass pairs and rep-
resents the optimization opportunities for a specific class of
programs. By leveraging multiple search spaces, the method
can effectively generalize to accommodate various program
types. Ultimately, a program can select and connect pass
pairs from the search space that align with its optimization
needs, thereby identifying an appropriate optimization se-
quence to improve performance.
4 Approach
The principal objective of our methodology is to facilitate
the rapid identification of near-optimal pass sequences for
previously unseen programs. The proposed method is illus-
trated in Figure 1. Initially, the dataset is divided into three
subsets: the training set, the validation set, and the test set.
In the first step, the chained synergy pass pairs for each pro-
gram in the training set are identified. These pairs representcombinations of compiler passes that work together in a way
that enhances the performance of compilation optimization
while also helping to control the search space [ 20]. These sets
are then clustered and merged to form 𝑘coresets of chained
synergy pass pairs. Next, the scores of these coresets for
each program in the training set are computed, resulting
in a score vector that represents the program’s probability
distribution across different coresets. After training the neu-
ral network, the model predicts the probability distributions
for the programs in the test set and selects the set with the
highest predicted probability for further exploration. Finally,
a search method is employed to find the near-optimal se-
quence within the selected set. To provide a more intuitive
demonstration of the effectiveness of our methodology, we
choose IR instruction reduction as the optimization target.

chained synergy pairs encompass both types of situations
described above.
4.2 Generating Core Chained Synergy Pass Pair Sets
We merge sets of chained synergy pass pairs for similar
programs based on the following observations: (1) From
Figure 3, it is evident that the effectiveness of different
chained synergy pairs varies. Some of the 𝐸(1,548) enu-
merated chained synergy pairs are effective across multiple
programs, while others are effective for only a few. Merg-
ing sets of chained synergy pairs for similar programs can
significantly reduce the number of highly similar sets. (2) In
our search for chained synergy pass pairs across differentprograms, we observed significant variability in the num-
ber of such pairs. Some programs have hundreds of chained
synergy pairs, while others have only a few. This variation
does not necessarily indicate a lack of optimization poten-
tial. Our strict definition aims to identify high-value pairs,
and some programs might require specific passes to uncover
additional chained synergy pairs. By merging these sets, we
can explore optimization opportunities more broadly across
similar programs. As programs undergo changes during the
search process with passes being applied, their characteris-
tics may shift, potentially revealing new chained synergy
pairs. Thus, similar programs, representing variations in fea-
tures, contribute to a more comprehensive search space that
may include novel chained synergy pairs.

program, despite its changes, may continue to exhibit similar

effective pass sequence within these sets. These strategies
range from simple random exploration to more sophisticated
approaches that leverage domain knowledge and machine
learning techniques. Specifically, we represent each chained
synergy pair set as a directed graph, where nodes represent
passes and directed edges represent chained synergy pass
pairs. We then use the following strategies to explore this
graph:
Random Strategy: The random strategy serves as the
simplest method for selecting optimal pass sequences by
randomly generating 100 different pass sequences within a
synergistic set. For each program, we randomly generate 100
pass sequences, with each sequence constrained to a maxi-
mum length of 20 and a minimum length of 4. The sequence
that yields the minimum IR instruction count among these
100 randomly generated sequences is selected as the final
result.
Greedy Strategy: The greedy strategy for optimizing
pass sequences works by iteratively selecting passes that
minimize the IR instruction count. Initially, we calculate the
IR instruction count for each pair of passes applied from
the synergistic set. The pass pair with the smallest IR in-
struction count is chosen as the starting point. From this

initial pair, the algorithm continues to expand the sequence
by adding passes that follow the last pass in the current se-
quence, aiming to further reduce the IR instruction count.
In each iteration, a candidate pass is added to the current
sequence to generate potential pass sequences. The pass that
results in the lowest IR instruction count is then appended
to the sequence. This process is repeated until the sequence
reaches a predefined maximum length of 20 passes. We keep
track of the intermediate results generated at each iteration,
and the final output of the greedy algorithm is the sequence
that achieves the minimum IR instruction count across all
iterations.
Genetic Algorithm: The genetic algorithm (GA) strat-
egy involves evolving a population of candidate solutions
through iterative processes of selection, crossover, and mu-
tation to identify an optimal pass sequence that minimizes
the IR instruction count. The key steps in our GA approach
include generating an initial population, selecting individu-
als based on fitness scores, performing crossover to combine
promising traits, and applying mutations to introduce di-
versity. Each step is designed to balance exploration and
exploitation, ensuring convergence toward high-quality so-
lutions.
•Initial Population Generation : The initial population is
generated using the results from the random strategy.
This approach provides a diverse set of starting points
that serve as potential solutions for the evolutionary
process.
•Selection and Crossover : The selection step identifies
the most suitable individuals for reproduction based
on their fitness scores, which are calculated as the per-
centage reduction in IR instruction counts compared
to the original program. Individuals are ranked, and a
selection rate is applied to choose parents. If two in-
dividuals share at least one common gene point, they
are selected for crossover; otherwise, the two highest-
scoring individuals are chosen. As illustrated in Figure
5a, the crossover process involves selecting a common
gene point (common node in the Figure) and swapping
segments of the two parents to create offspring. For
two sequences 𝑃𝑖𝑃𝑞𝑃𝑘𝑃𝑚and𝑃𝑖𝑃𝑗𝑃𝑘𝑃𝑛, the common
node is𝑃𝑘. During the crossover, the paths following
𝑃𝑘in both sequences are swapped, resulting in the
offspring sequences 𝑃𝑖𝑃𝑞𝑃𝑘𝑃𝑛and𝑃𝑖𝑃𝑗𝑃𝑘𝑃𝑚.
•Mutation : The mutation process, depicted in Figure 5b,
introduces additional diversity into the population by
modifying existing solutions. A mutation operation is
triggered at a specified mutation rate, where a muta-
tion point is chosen—typically a node with multiple
successor nodes. From this point, a random outgoing
edge is selected, and unvisited subsequent nodes are
added to form a new mutated path. This operation
continues until no further new nodes can be found.For example, as shown in Figure 5b, in the sequence
𝑃𝑖𝑃𝑞𝑃𝑘𝑃𝑚, nodes𝑃𝑖,𝑃𝑞,𝑃𝑘each have an out-degree
greater than 1 and are considered potential mutation
points. If𝑃𝑞is randomly selected as the mutation point,
the sequence is mutated to 𝑃𝑖𝑃𝑞𝑃𝑗𝑃𝑘. The resulting mu-
tated path is then added back into the population as a
new candidate solution.
Reinforcement Learning: The reinforcement learning
(RL) strategy utilizes an agent-based approach to search for
optimal pass sequences within a synergistic set 𝑆predicted
by a supervised model for a given program 𝑃. The key idea
is to employ an action masking mechanism to dynamically
constrain the search space, guiding the agent toward con-
structing valid and effective sequences. The agent starts by
randomly choosing a chained synergy pair (𝐴,𝐵)from the
set𝑆. After selecting this pair, the search space is dynami-
cally updated to form a new subset 𝑆𝑠𝑡𝑒𝑝 1, which contains
all chained synergy pairs beginning with 𝐵. All pairs not in
𝑆𝑠𝑡𝑒𝑝 1are masked to restrict future choices. The agent then
selects the next chained synergy pair from this updated set.
For example, if the agent chooses (𝐵,𝐷), the search space
is updated again to form 𝑆𝑠𝑡𝑒𝑝 2, containing only pairs start-
ing with𝐷. This process continues iteratively, constructing
the pass sequence step-by-step until a maximum number of
steps is reached or no further valid actions are available. We
use the 56 features in Autophase to represent the state space,
capturing the state of the IR. The reward at each step is the
reduction ratio of the IR instruction count compared to the
previous step. The total reward is the cumulative reduction
ratio across all steps, measuring the effectiveness of the pass
sequence.
These search strategies—random, greedy, genetic algo-
rithm and reinforcement learning—offer different approaches
to finding efficient pass sequences from a synergistic set. Sim-
ple methods like random and greedy provide straightforward
solutions, while the genetic algorithm and reinforcement
learning use more sophisticated techniques to explore the
search space. By employing these search strategies in con-
junction with our chained synergy pass pair set, we aim to
optimize programs and reduce the number of IR instructions.

5.3 Ablation Experiment
5.3.1 Connecting vs. Non-Connecting. In our method,
the connection of chained synergy pairs must adhere to the
rule that the pre-pass of each subsequent pair matches the
post-pass of the previous pair. To assess the impact of disre-
garding this connectivity rule, we conducted a comparative
analysis. We reselected 100 random sequences that do not
adhere to the connectivity rule for each coreset to compute
scores and retrain the model. After retraining, we compared
the results of random strategies that also disregarded the
connectivity rule with those of the previous method.
Table 6 shows that the reduction in IR instruction count
achieved without following the connectivity rule is generally
weaker compared to when the rule is enforced. The great-
est difference in effectiveness between the two methods is
observed in the NPB benchmark. As illustrated in Figure 6,
adhering to the connectivity rule tends to lead to the discov-
ery of more valuable pass sequences. This is because, without
the connectivity rule, additional considerations are required
regarding the influence of the pre-pass of the next pair on
the post-pass of the previous pair, which reduces the ad-
vantageous scenarios for chained synergy pair connections.
This demonstrates the importance of utilizing connectiv-
ity in chained synergy pairs during our search within the
synergistic set.

6 Conclusion
This paper presents a novel approach for finding near-
optimal pass sequences for compiler optimization. Our ap-
proach leverages the synergistic relationships between com-
piler passes by identifying and clustering chained synergy
pass pairs to create a beneficial search space. We then uti-
lize a supervised learning model to select a suitable chained
synergy pass pair set for unseen programs based on their
features, effectively generalizing across different program
domains. Finally, we employ diverse search strategies, in-
cluding random walk, greedy algorithm, genetic algorithm
and reinforcement learning, to explore the selected set and
discover the near-optimal pass sequence for a given program.
Our experimental results on ten benchmark datasets demon-
strate that our approach achieves state-of-the-art perfor-
mance in reducing IR instruction count compared to existing
techniques. Notably, our approach consistently outperforms
other single-program search algorithms in reducing the code-
size, highlighting the effectiveness of our chained synergy

A.5 Experiment workflow
After downloading the Docker image and creating a con-
tainer, enter the container:
cd /home/cfsat/
conda activate cgo_paper323
The following experimental scripts contain only the experi-
mental procedures and results of the methods proposedin the paper, and do not contain the experiments and
results of the other techniques compared .
•About Table 4 main results :
–Random Strategy:
cd ./shell/
python Phase4_2_geannvp_100test.py
–GA Strategy:
cd ./shell/
python Phase4_2_geannvp_GAtest.py
–Greedy Strategy:
cd ./shell/
python Phase4_2_geannvp_Greedytest.py
–RL Strategy:
python run_rl.py --filepath XXX
XXX is the specific file path. For example, python
run_rl.py –filepath cbench-v1/cbench-v1_adpcm.ll .
•About Table 5 main results :
–Random Strategy:
python run_codesize_Random.py \\
--dataset XXX
–GA Strategy:
python run_codesize_GA.py --dataset XXX
XXX can be blas-v0, cbench-v1, mibench-v1, npb-v0,
opencv-v0 .
If the user wants to reproduce the whole process, please
follow the PhaseX_x script inside the shell folder for a step-
by-step reproduction.
A.6 Evaluation and expected results
For the experimental results of Table 4, the Random, Greedy,
and GA strategies are saved to ./csfat/output/ folder as Phas
e4_geannvp _100Result.csv ,Phase4 _geannvp _GAResult.csv ,
andPhase4_geannvp_GreedyResult.csv , respectively.
The RL policy is terminal output and the results of a single
file run are obtained via the episode_reward_max output by
rllib. In the experimental results in Table 5, the experimental
outputs for the random and greedy strategies are also at
the terminal, where Current File is the benchmark that is
currently being tuned, and Mean is the total average codesize
reduction percentage (compared to Oz) for all benchmarks
that have been tuned up to now. The last Mean output is the
average percentage reduction in codesize for all benchmarks
in this dataset compared to Oz.
A.7 Experiment customization
How to try other compiler versions? Copy the other
versions of the compiler tools (opt, clang++, llvm-size) to
the./cfsat/llvm_tools/ folder and add the action file for the
corresponding version of the compiler’s opt pass to /cf-
sat/LLVMEnv/actionspace/ . But note that the dataset we pro-
vide is the LLVM version 10.0.0 .llfile. So users need to
provide a compatible dataset when you change the compiler
version.
